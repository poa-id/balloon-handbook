# Continous Integration (CI)

La calidad del código es indispensable para nosotros. Una de nuestras reglas más estrictas que seguimos es la Integración Continua. 
Tener tests automatizados introduce una capa extra de confianza, y permite que el equipo detecte los problemas al integrar cambios, tales como bug fixes y componentes que estén fallando.

En cada *Git commit*, se corren tests automáticos en los archivos cambiados. En cada Pull Request, se corren tests para todo el código, que nos ayudan a identificar si algo se rompió en los últimos commits.

Cuando no se encuentra ningun problema, se verifica manualmente el código y luego ocurre el Merge.

Se trata de reducir los riesgos.
